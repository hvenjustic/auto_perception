# 自动感知网站变化系统需求（target）

## 1. 背景与目标

构建一个自动感知网站变化的工具，提供 Web API 供 `clawdbot` 调用。  
系统需要支持通过 API 导入待监测网站配置，并在检测到变化后输出摘要。

核心目标：
1. 低成本、高效率地发现网站更新。
2. 输出可供机器人消费的结构化变更结果。
3. 对变更内容生成可读摘要。

## 2. 总体架构

采用“三层感知法”，避免爬取所有子页面，先筛选再深入：

1. 第一层：入口监控（低成本筛选）
2. 第二层：深度扫描（按需触发）
3. 第三层：差异提取（智能摘要）

## 3. 详细功能需求

### 3.1 第一层：入口监控

目标：快速发现“可能有变化”的页面候选集合。

1. RSS/Atom 订阅检测
- 优先检测 `atom.xml` / `rss.xml`。
- 通过条目更新时间与历史记录比对，识别新增或更新条目。

2. Sitemap 监控
- 检测 `sitemap.xml`。
- 对比 `<lastmod>` 时间戳，定位更新子页面。

3. Index 页面结构对比
- 监控首页或列表页 HTML 结构变化（例如新闻列表首行）。
- 若结构变化，则将相关链接加入候选页面集合。

### 3.2 第二层：深度扫描

目标：对候选页面做精准变更判断。

1. 指纹对比（Hashing）
- 对页面核心内容计算哈希（MD5 或 SHA-256）。
- 新旧哈希不一致即判定页面内容变更。

2. 关键 DOM 提取
- 不监控整页。
- 重点提取 `article`、`main` 或指定 `div`。
- 排除页脚、广告、随机推荐等噪声区域。

### 3.3 第三层：差异提取

目标：将技术变化转为业务可读摘要。

1. 调用 LLM 对比新旧内容。
2. 输出变更摘要，包含“更新了什么”的核心信息。

## 4. API 能力要求

1. 配置导入 API
- 支持通过 API 导入待监测网站地址与相关配置。

2. 检测触发 API
- 支持触发监测流程并返回本次检测结果。

3. 变更查询 API
- 支持查询历史变更记录与摘要。

## 5. 非功能要求

1. 效率：优先低成本入口检测，减少无效深爬。
2. 可扩展：支持新增站点与规则配置。
3. 可集成：API 设计适配 `clawdbot` 调用链路。

## 6. 验收标准

1. 可通过 API 成功导入监测网站配置。
2. 系统可检测入口变化并触发候选页面深度扫描。
3. 系统可识别页面内容变更并输出摘要。
4. 外部调用方可通过 Web API 获取变更结果与摘要信息。
